## 🎯 設計原則（v2 更新）

| 原則 | 說明 |
|------|------|
| **緩存與確定分離** | 未完成收款 / 未完成服務 = 緩存（buffer）。全部完成 = 確定收入。兩者清楚分開 |
| **訂單為本** | 每一筆 client_service = 一個訂單。訂單有總額、有已收、有剩餘次數。收錢跟訂單走 |
| **記錄為主** | 所有變動記到 activity_log，不改不刪。店長自己判斷 |
| **自由輸入** | 每次收幾多錢、做幾多次，店長自由決定，系統唔自動計算 |
| **收入記錄 = 訂單記錄** | 下方列表每個 row 係一筆收錢動作（來自訂單），不是會計帳 |

## ② 緩存概覽 + 確定收入

### 緩存中訂單（Buffer）
任何未完全完成嘅 client_service = 緩存：
- 仲有 remaining_sessions > 0（未做完服務）
- OR 已收金額 < 總額（未收足錢）

### 確定收入（Confirmed Revenue）
已完成 + 已收足錢嘅訂單總金額。

### 互動
- 點擊「緩存中訂單」→ 跳轉收入紀錄頁（篩選緩存狀態）
- 點擊「確定收入」→ 跳轉收入紀錄頁（篩選已完成）
