## 🎯 設計原則

| 原則 | 說明 |
|------|------|
| **記錄為主** | 所有變動記到 activity_log，不改不刪。店長自己看記錄判斷差異 |
| **一眼知重點** | 老闆每日開 App 3 秒就看到今天忙不忙、賺多少、有冇麻煩嘢 |
| **不自動修正** | 退回後交易標 VOID 但不改結算金額。店長看標記，人手處理現金差異 |
| **簡單直接** | 不需要 T 字帳、會計回沖。線性記錄，清楚即可 |

## ① 今日實況 (Real-time Snapshot)

### 目的
老闆打開 App 第一眼就知道今日舖頭狀況。

### 顯示內容

| 指標 | 資料來源 | 說明 |
|------|---------|------|
| 總預約數 | `appointments` WHERE `appointment_date` = today | |
| 已完成 | status = `attended` | |
| 待處理 | status = `confirmed` | 尚未到 / 尚未標記 |
| 失約 | status = `no_show` | 紅色高亮如果 > 0 |
| 待處理退回 | status = `attended` 但曾被提問？ | 目前無此狀態，可省略 |

### 互動
- 點擊任何數字 → 跳轉到對應的篩選列表（例如點擊「失約 1」→ 今日預約列表篩選 no_show）

## ② 今日收入快覽

### 目的
老闆不用等到夜晚結算，隨時知道做到幾多錢。

### 顯示內容

| 指標 | 資料來源 | 說明 |
|------|---------|------|
| 總收入 | `payment_transactions` WHERE `transaction_date` = today (含 VOID) | |
| 現金 | WHERE `payment_method` = 'cash' | |
| 信用卡 | WHERE `payment_method` = 'card' | |
| 轉賬 | WHERE `payment_method` = 'transfer' | |
| ⚠️ VOID 筆數 | WHERE `remarks` LIKE '%VOID%' | 小字灰底顯示，不扣減總額 |

### 互動
- 點擊金額卡片 → 跳轉到每日結算頁面看明細
- VOID 筆數可點擊展開查看哪些交易被退回

### 設計哲學
> 系統不自動扣減 VOID 交易的金額。總收入照樣顯示所有 payment 的 SUM。店長看到 VOID 筆數後，自己判斷是否需要調整實收現金。結算鎖定時在備註欄寫低差異。

## ③ 客戶活躍度 / 沉睡警報

### 目的
提醒老闆做客戶維繫，唔好等到客走晒先發現。

### 顯示內容

| 指標 | 資料來源 | 說明 |
|------|---------|------|
| 活躍客戶 | `clients` WHERE `last_visit_date` ≥ 30 天前 | |
| 沉睡客戶 | `dormant_clients` View (>90 天) | 紅色數字，點擊跳轉沉睡客列表 |
| 新客戶 (本月) | `clients` WHERE `created_at` 在同月 | |

### 互動
- 點擊沉睡客戶數字 → `/dormant` 頁面
- 點擊新客戶數字 → 客戶列表頁（篩選本月新加入）

## ④ 療程熱度排行

### 目的
老闆知道邊個療程最好賣，可能調整推廣策略或 pricing。

### 顯示內容

| 指標 | 資料來源 | 說明 |
|------|---------|------|
| 療程名稱 + 扣減次數 | `activity_log` WHERE `action_type` = 'deduct_service' AND `created_at` ≥ 30 天前，GROUP BY `treatment_name`，COUNT，ORDER BY COUNT DESC | 簡單 Bar |

## ⑤ 員工表現簡表 (可選，僅店長可見)

### 目的
老闆了解員工忙碌程度，分配資源或獎勵。

### 顯示內容

| 指標 | 資料來源 | 說明 |
|------|---------|------|
| 員工名 | `profiles` WHERE `role` = 'staff' | |
| 本月 completed 預約數 | `appointments` WHERE `status` = 'attended' AND `staff_id` = ... AND `appointment_date` 在同月 | |
| 本月收入貢獻 | `payment_transactions` WHERE `staff_id` = ... AND `transaction_date` 在同月 | |

### 注意
- ⚠️ 此區塊僅店長可見，員工看不到
- 不顯示個別員工扣數明細，只顯示總計

## ⑥ 待辦事項 / 異常提示

### 目的
防止漏收錢、客投訴過期、麻煩客頻繁甩底。

### 顯示內容

| 警報類型 | 觸發條件 | 處理方式 |
|---------|---------|---------|
| 🔴 失約 2 次以上客戶 | `no_show_count` ≥ 2 | 列出客戶名 + 失約次數，店長決定是否封鎖 |
| 📅 套票 7 日內到期 | `client_services.expiry_date` BETWEEN today AND today+7 | 提醒店長通知客戶來用 |
| 🔙 有待確認退回 | 無自動偵測，店長自行注意 | |
| 💰 有待處理退款 | `refunds` 表中未處理？ | 目前無 pending 狀態 |

## 🧭 MVP 優先級

### 第一階段 (必須有)

| 區塊 | 理由 |
|------|------|
| **① 今日實況** | 最基本，取代紙本日記 |
| **② 今日收入快覽** | 取代人手計數 |
| **③ 沉睡客戶數** (僅數字 + 快捷入口) | 最重要嘅客戶維繫指標 |
| **⑥ 異常提示** (失約 + 套票到期) | 防止漏嘢 |

### 第二階段 (有時間加)

| 區塊 | 理由 |
|------|------|
| ④ 療程熱度 | 幫助推廣決策 |
| ⑤ 員工表現 | 管理需要，但要小心設計 |
