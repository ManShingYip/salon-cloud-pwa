## 關鍵設計規則

### 兩個框架（頂部 summary cards）
- 🟡 **緩存中**：任何 remaining_sessions > 0 OR SUM(已收) < total_price 嘅訂單
- 🟢 **確定收入**：remaining = 0 AND SUM(已收) >= total_price 嘅交易
- 每個 framework card 顯示筆數同金額
- 點擊可以 filter 下方 list

### 下方 List（一單一記錄）
- **每一行 = 一筆 payment_transactions**（收錢動作）
- 顯示：客戶名、療程名、已收金額/總額（分期進度）、完成度 bar、收款方式、日期
- 如果係分期：顯示 `$1,500/$5,000 ███░░░░ 30%`
- 如果係完成：顯示 `$380/$380 ██████ 100%` + 綠色標記
- **服務日期跟治療流程** — 用 payment_transactions.transaction_date（即係收錢當日）

### 篩選機制
- 日期範圍
- 狀態篩選：全部 / 緩存中 / 確定收入 / 已退款
- 客戶搜尋（姓名或電話）
- 收款方式篩選

## 資料查詢邏輯

### 緩存中訂單
```sql
-- 從 client_services 搵出未完成嘅訂單
SELECT cs.*, c.name, c.phone, t.name as treatment_name,
  COALESCE(SUM(pt.amount), 0) as total_paid
FROM client_services cs
JOIN clients c ON cs.client_id = c.id
JOIN treatments t ON cs.treatment_id = t.id
LEFT JOIN payment_transactions pt ON pt.client_id = cs.client_id AND pt.treatment_id = cs.treatment_id
WHERE cs.status = 'active'
  AND (cs.remaining_sessions > 0 OR cs.total_price > COALESCE(SUM(pt.amount), 0))
GROUP BY cs.id, c.name, c.phone, t.name
```

### 確定收入交易
```sql
-- 已完成嘅 payment_transactions（所屬訂單已完成）
SELECT pt.*, c.name, c.phone, t.name as treatment_name
FROM payment_transactions pt
JOIN clients c ON pt.client_id = c.id
JOIN treatments t ON pt.treatment_id = t.id
WHERE pt.remarks NOT LIKE '%VOID%'
  -- 所屬 client_service 已完成
  AND EXISTS (
    SELECT 1 FROM client_services cs
    WHERE cs.client_id = pt.client_id
      AND cs.treatment_id = pt.treatment_id
      AND cs.remaining_sessions = 0
      AND cs.status = 'expired'
  )
ORDER BY pt.transaction_date DESC
```

## 前端實作步驟

1. **加篩選列** — 日期範圍 + 狀態下拉 + 客戶搜尋 + 付款方式
2. **頂部雙框架卡** — 緩存統計 + 確定收入統計
3. **下方 List** — 每個 row 顯示進度 bar + 狀態標記
4. **點擊框架卡 filter** — 點擊緩存卡 → 只顯示緩存中，點擊確定卡 → 只顯示已完成
5. **資料 query** — 從 client_services + payment_transactions JOIN 出訂單狀態
